# Regras do Projeto Venus Engine

1. Nunca alterar arquivos fora deste projeto.
2. Não criar integrações externas sem aprovação explícita no prompt da tarefa.
3. Toda nova tela deve priorizar mobile-first.
4. Toda funcionalidade deve ter foco em UX premium, clareza e redução de fricção.
5. Nunca gerar arquitetura confusa ou duplicada.
6. Sempre preferir componentes reutilizáveis.
7. Sempre separar experiência B2C e B2B.
8. Toda recomendação de moda deve considerar:
   - emocional
   - paleta
   - biotipo
   - visagismo
   - acessórios
   - contexto de uso
9. Toda sugestão visual precisa terminar em ação clara.
10. Nunca mexer no projeto InovaCortex a partir deste workspace.
