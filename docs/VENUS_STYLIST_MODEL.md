# Venus Stylist Model

This document defines the operating model for Venus as a high-end stylist engine tied to the store catalog.

## Product goal

The product should feel like a senior personal stylist that:
- reads the person from photo and intake
- selects only catalog items that fit the profile
- explains why the choice works
- turns the result into WhatsApp action
- turns the shared image into acquisition for the store

## Ideal flow

1. Intent calibration
- Capture the line of styling first.
- Capture the image goal.
- Capture the main pain and confidence level.

2. Visual reading
- Read face photo and body photo with AI.
- Derive visual direction, palette, contrast, and visagism.
- Keep the questionnaire as support, not the center.

3. Catalog curation
- Rank real store products by role, direction, color, fit, use case and formality.
- Build three looks:
  - base
  - support
  - statement
- Always explain why each piece is in the look.

4. Result and conversion
- Show the essence captured.
- Show the look hierarchy.
- Show what to avoid.
- Offer WhatsApp handoff.
- Offer sharing/posting with the store and Venus signature.

5. Social loop
- The shared post should already carry the store branding.
- The shared post should make people want to test the store experience.
- The user should feel that the store has a premium stylist engine behind it.

## Catalog data required per product

Minimum data:
- name
- category
- style
- type
- primary color
- price range
- image URL

Stylist data preferred:
- style direction
- style tags
- category tags
- fit tags
- color tags
- target profile
- use cases
- occasion tags
- season tags
- body effect
- face effect
- visual weight
- formality
- catalog notes

## Screen order

The customer experience should follow this order:

1. Home
- Promise the stylist experience.
- Show the premium tone quickly.

2. Intent calibration
- Set styling direction.
- Set image goal.
- Capture the main pain.

3. Lifestyle
- Read environments and purchase DNA.
- Identify buying behavior.

4. Colors
- Capture what the person likes and avoids.
- Capture metal preference.

5. Body / face
- Capture fit and face lines.
- Keep it practical and quick.

6. Visual scan
- Show the AI is reading the person, not only collecting answers.

7. Processing
- Make the AI feel active and alive.

8. Result
- Deliver the essence.
- Deliver the looks.
- Deliver the WhatsApp and share loop.

9. Product modal
- Explain the piece like a stylist.
- Show the rationale, tags and use case.

## Quality bar

The product is successful when the user feels:
- "It understood me"
- "This look is for me"
- "This feels like a real stylist"
- "I want to share this"
- "I want to test the store"
